# Pipeline — Briefs técnicos

Estos archivos no son el prompt inicial. Definen alcance, tandas, arquitectura, riesgos, prompts
auxiliares por rol y criterios de aceptación de cada fase.

Para empezar una fase, usar el archivo correspondiente de `docs/pipeline/prompts/`. Ese prompt
obliga al agente a leer el brief adecuado. Al cerrar, el resultado se documenta en `docs/reports/`.

```text
prompt de ejecución → brief técnico → implementación/checkpoints → reporte de evidencia
```
