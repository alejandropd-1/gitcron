# Contexto
Tengo una interfaz de usuario en React/Next.js generada por Stitch para un clon de GitKraken. Actualmente es solo la maqueta visual. Necesito implementar la lógica funcional conectando esta UI con un backend local usando la librería `simple-git`. La aplicación correrá en un entorno de escritorio (Electron/Tauri), por lo que asume que existe un puente IPC (Inter-Process Communication) para ejecutar comandos de Node.js.

# Objetivo
Generar los manejadores de estado (state hooks) y las funciones controladoras (event handlers) para dar vida a las siguientes características clave de la interfaz, mapeando las acciones del usuario a comandos reales de Git.

# Funcionalidades requeridas

**1. Operaciones del Menú Contextual (Ramas y Commits):**
* **Merge:** Función para fusionar una rama seleccionada en la rama actual (`git merge <selected-branch>`). Debe manejar posibles conflictos devolviendo un estado de error a la UI.
* **Revert:** Función para revertir un commit específico seleccionado en el árbol (`git revert <commit-hash>`). 

**2. Área de Staging y Commit:**
* **Input de texto:** Manejador de estado para el área de texto del mensaje de commit.
* **Ejecutar Commit:** Función que tome el texto del input y los archivos en *stage* para ejecutar `git commit -m "mensaje"`.

**3. Gestión de Cambios (Stash y Discard):**
* **Stash:** Función vinculada al botón "Stash" de la barra superior para guardar los cambios locales no commiteados (`git stash`).
* **Descartar cambios (Trash can / Revertir archivo):** Función vinculada al icono de "papelera" en la lista de archivos modificados. Debe ejecutar `git checkout -- <file_path>` o `git restore <file_path>` para eliminar las modificaciones locales de ese archivo específico y devolverlo a su estado original en el HEAD de la rama.

**4. Estructura de Datos:**
Proporciona la estructura del estado global (ej. Zustand o Context API) recomendada para mantener sincronizado:
* La rama actual.
* La lista de archivos modificados (untracked, modified, deleted).
* El historial de commits para renderizar el grafo.

# Entregables
1.  El código de los `useEffect` o funciones que llamarán al backend (ej. `window.api.gitCommand(...)`).
2.  La lógica de los componentes de React para los botones y menús mencionados, mostrando cómo se conectan a estas funciones.
